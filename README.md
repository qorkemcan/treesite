# ProTreeTrim Website

ProTreeTrim is an Astro-based static website for Florida tree service dispatch, homeowner education, and local service discovery.

The site combines high-intent service pages, Florida homeowner tools, county hubs, blog content, partnership pages, contact flows, and lightweight audio guides.

## Tech Stack

- Astro 5
- Vercel adapter for deployment
- Static-first architecture
- CSV/data-driven city and county content
- Markdown/MDX-style blog content under `src/content/blog/`
- Static assets served from `public/`

## Audio Guide Pilot

Selected high-intent pages include short audio guides for accessibility, user trust, and homeowner guidance.

Current audio guide areas include:

```txt
/tools/emergency-tree-safety-checklist/
/tools/florida-tree-care-advisor/
/tools/florida-problem-tree-guide/
/services/emergency-response/
/services/tree-removal/
/services/stump-grinding/
/county/volusia/
```

Implementation notes:

- Shared component: `src/components/AudioGuideBlock.astro`
- MP3 files live in: `public/audio/`
- Audio players should use `preload="none"` so files are not downloaded until the visitor presses play.
- Keep new audio guides short and only add them to high-intent pages where they improve the user experience.

## Development Commands

Install dependencies:

```bash
npm install
```

Start local development:

```bash
npm run dev
```

Build the production site:

```bash
npm run build
```

Preview production build locally:

```bash
npm run preview
```

The build command runs sitemap generation before Astro build:

```bash
node generate-sitemaps.mjs && astro build
```

## Deployment

The project is deployed on Vercel. Pushing to the `main` branch triggers deployment.

Recommended deployment flow:

```bash
git status --short
npm run build
git add <changed-files>
git commit -m "Describe change"
git pull --rebase origin main
git push origin main
```

After deployment, verify key changed URLs using the browser or `curl`.

Example:

```bash
curl -I https://www.protreetrim.com/tools/florida-tree-care-advisor/
```

Expected result:

```txt
HTTP/2 200
```

## Sitemap Notes

Sitemaps are generated by `generate-sitemaps.mjs` and stored under `public/`.

If `npm run build` modifies sitemap files unexpectedly, review the changes before committing. Do not commit sitemap noise unless the URL/content changes are intentional.

## Repo Hygiene Rules

Avoid committing temporary install, preview, and ZIP files unless they are intentionally part of the project history.

Common temporary files/folders that should usually stay out of commits:

```txt
*_preview/
*_input.zip
*-input.zip
*-install.zip
protreetrim-*.zip
```

Useful install notes may remain in the repo for now as `_INSTALL_NOTES_*.md`. If they become too numerous, move them into a documentation folder such as:

```txt
docs/install-notes/
```

## Current Project Priorities

- Keep high-intent Florida service pages clean, fast, and conversion-focused.
- Preserve crawlability and indexability for tools, services, counties, and high-quality blog content.
- Use audio guides selectively as a user-experience layer, not as bulk content.
- Avoid unnecessary technical experiments that could risk indexing, routing, or conversion paths.
- Monitor Google Search Console before making large programmatic expansions.
